# Quick Start Guide

This guide will help you get up and running with FairyChat's core features.

## Basic Configuration

After installation, FairyChat generates a configuration file at `plugins/FairyChat/config.yml`. Here's how to set up a basic chat format:

### Customizing Chat Format

The chat format controls how messages appear in chat. FairyChat supports MiniMessage formatting:

```yaml
format:
  # Default chat format for all players
  default: "<fc_luckperms_prefix><sender_name><reset> <gray>»</gray> <message>"
  
  # Per-group formats (requires LuckPerms)
  groups:
    admin: "<red>[Admin]</red> <fc_luckperms_prefix><sender_name><reset> <gray>»</gray> <message>"
    moderator: "<blue>[Mod]</blue> <fc_luckperms_prefix><sender_name><reset> <gray>»</gray> <message>"
    vip: "<gold>[VIP]</gold> <sender_name><reset> <gray>»</gray> <message>"
```

### Built-in Placeholders

FairyChat includes several built-in placeholders:

| Placeholder | Description |
|-------------|-------------|
| `<sender_name>` | The player's username |
| `<sender_displayname>` | The player's display name |
| `<server_name>` | Server name from config |
| `<message>` | The chat message content |

### LuckPerms Placeholders

If you have LuckPerms installed, you can use these placeholders:

| Placeholder | Description |
|-------------|-------------|
| `<fc_luckperms_prefix>` | Player's prefix |
| `<fc_luckperms_suffix>` | Player's suffix |
| `<fc_luckperms_username_color>` | Username color from meta |
| `<fc_luckperms_message_color>` | Message color from meta |

### Using PlaceholderAPI

For PlaceholderAPI placeholders, use the special format:

```yaml
format:
  default: "<papi:'player_name'> <gray>»</gray> <message>"
```

## Essential Commands

Here are the most commonly used commands to get you started:

### Broadcasting Messages

Send a message to all players across all servers:

```bash
/broadcast <message>
```

**Permission:** `fairychat.command.broadcast`

**Example:**
```bash
/broadcast <red>Server will restart in 5 minutes!</red>
```

<Note>
  Players with `fairychat.feature.minimessage` permission can use MiniMessage formatting and color codes in broadcasts.
</Note>

### Private Messaging

Send a private message to another player:

```bash
/pm <username> <message>
```

Reply to the last private message:

```bash
/r <message>
```

**Features:**
- Works across multiple servers (with Redis configured)
- Tab completion for player names
- Respects player ignore lists

### Managing Ignored Players

Toggle ignoring a specific player:

```bash
/ignore <username>
```

Players with `fairychat.bypass.ignore` permission cannot be ignored.

### Clearing Chat

Clear the chat for all players or a specific player:

```bash
/clearchat [player]
```

**Permission:** `fairychat.command.chatclear`

### Plugin Management

Reload the configuration without restarting:

```bash
/fairychat reload
```

**Permission:** `fairychat.command.fairychat.reload`

## Testing Your Setup

<Steps>
  <Step title="Test Chat Format">
    Send a message in chat and verify that your custom format appears correctly.
    
    If you have per-group formats configured, test with players from different permission groups.
  </Step>
  
  <Step title="Test Private Messages">
    Use `/pm <player> <message>` to send a private message to another player.
    
    Then use `/r <message>` to test the reply functionality.
  </Step>
  
  <Step title="Test Mentions">
    Mention another player using `@username` in chat.
    
    The mentioned player should receive a notification sound.
  </Step>
  
  <Step title="Test Broadcasts">
    Use `/broadcast <message>` to send a server-wide announcement.
    
    All online players should see the message.
  </Step>
</Steps>

## Important Permissions

Here are key permissions you'll want to configure:

| Permission | Description |
|------------|-------------|
| `fairychat.feature.minimessage` | Allows MiniMessage & color codes in chat |
| `fairychat.feature.displayitem` | Allow players to show items with `<item>` |
| `fairychat.bypass.cooldown` | Bypass chat cooldown |
| `fairychat.bypass.similarity` | Bypass similarity check |
| `fairychat.bypass.ignore` | Cannot be ignored by others |
| `fairychat.notify-update` | Receive update notifications on join |
